# video-use 프로젝트 컨텍스트

## 이 프로젝트는

오픈소스 영상 편집 도구. 폴더에 원본 영상을 넣고 에이전트에 "이거 편집해줘"라고 하면 `edit/final.mp4`가 나오는 시스템.

핵심 동작은 `SKILL.md`에 정의되어 있고, 실제 편집 스크립트는 `helpers/` 안에 있음. `skills/manim-video/`는 오버레이 애니메이션 생성용 서브 스킬.

영상 편집 작업 자체에 들어갈 때는 **반드시 `SKILL.md`를 먼저 읽을 것** — 12가지 production-correctness 규칙이 거기 있고, 이건 협상 불가.

## 커뮤니케이션 규칙

- 전문 용어는 처음 등장 시 짧게 풀어서 (예: "스테이징 = 커밋 후보로 골라둔 상태")
- 명령어를 보여줄 때는 한 줄 주석으로 무엇을 하는지 설명
- "당연히 알 것" 전제 금지
- 위험한 명령(`rm`, `git reset --hard`, `force push`, `.env` 처리 등)은 사전 설명 + 사용자 승인 후 실행
- 결과물은 가능하면 파일로 저장 (채팅으로만 끝내지 않음)

## 보안 규칙

- `.env`, `.env.local` 등 시크릿 파일은 **읽기/쓰기 모두 차단**되어 있음 — 사용자가 직접 처리
- `.env.example`에는 절대 실제 값을 넣지 말 것 (오직 placeholder)
- ElevenLabs API 키는 `.env`의 `ELEVENLABS_API_KEY`에 보관

## 작업 워크플로우

큰 변경(여러 파일 수정, 폴더 정리 등)은 **계획부터 보여주고 승인 후 실행**.

1. 읽기·조사·분석을 먼저
2. 계획(`plan.md`) 합의
3. 실행
4. 검증
5. 필요 시 `handoff.md` 갱신

## 자주 쓰는 명령

```bash
uv sync                 # Python 의존성 동기화
brew install ffmpeg     # 영상 처리 필수
```

## 이 프로젝트의 외부 환경

- `~/.claude/skills/video-use` 가 이 디렉토리로 심볼릭 링크됨 (Claude Code 스킬로 등록)
- 영상 작업 시 사용자는 영상 폴더에서 `claude` 실행 → 출력은 `<영상폴더>/edit/`에 떨어짐
- 이 디렉토리(`~/Developer/video-use`) 자체는 **도구·코드 저장소**이며 영상 작업 폴더가 아님
